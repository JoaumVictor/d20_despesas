**Erro:** `TypeError: Cannot read property 'ErrorBoundary' of undefined` em
expo-router `useScreens.js` (fromImport) ao iniciar o app.
**Causa raiz:** `AuthContext.tsx` chamava `GoogleSignin.configure()` no topo do
módulo (side-effect de import). O `@react-native-google-signin/google-signin`
(v16, config-plugin) não existe no Expo Go e não há prebuild/dev-client, então o
`configure()` estourava na avaliação do módulo → o root `_layout` não carregava →
`loadRoute()` retornava `undefined` → expo-router quebrava.
**Hipóteses testadas:** (1) export default faltando → ❌; (2) import circular/barrel
→ ❌; (3) `Stack.Protected` sem suporte → ❌; (4) side-effect de topo com módulo
nativo ausente no Expo Go → ✅ confirmada.
**Arquivos tocados:**
- app/src/features/auth/AuthContext.tsx (modificado)
**Fix aplicado:** Removido `GoogleSignin.configure()` do topo do módulo; movido para
`configureGoogleSignin()` chamado dentro do `AuthProvider` via `useEffect`, com
try/catch e aviso claro. Falha do módulo nativo agora fica localizada (não derruba
as rotas). `tsc --noEmit` limpo.
**Pendência:** Google Sign-In real só funciona em development build
(`npx expo run:android`), não no Expo Go.
**Status:** ✅ Resolvido (boot crash); ⚠️ login exige dev build

registro está em 0 e o codegen C++ da Nova Arquitetura (RN 0.86) do
`react-native-gesture-handler` gera um `.o` cujo path embute o caminho completo do
projeto duas vezes, estourando 260 chars. Projeto em `C:\Users\Victor\projects\
PESSOAL\d20_despesas\app` (base 49 chars). gesture-handler é transitiva (expo-router
/react-native-screens); newArch não é desligável no RN 0.86.
**Hipóteses testadas:** (1) erro de código/config → ❌; (2) path >260 com LongPaths
desligado → ✅ confirmada (mensagem do ninja).
**Arquivos tocados:** nenhum (mudança de ambiente).
**Fix aplicado (final):** Comparado com projeto que builda na mesma máquina
(PLATHANUS/caars, gesture-handler 2.30.1 — não tem o arquivo problemático). Causa
específica: `react-native-gesture-handler 3.0.2` (transitivo, peer do expo-router,
não usado pelo app) traz um `.cpp` de codegen com path que estoura 260. Solução:
`expo.autolinking.exclude: ["react-native-gesture-handler"]` no package.json +
limpeza de `android/app/.cxx`. Build passou: `BUILD SUCCESSFUL`, app-debug.apk gerado.
LongPaths/admin não foram necessários.
